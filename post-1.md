# Post 1: Introduction to Programming - My Computer Engineering Journey

Stepping into the University of Engineering and Technology (UET) Lahore as a Computer Engineering student has been one of the most exciting and challenging experiences of my academic life. In this first semester, our core focus shifted from just being basic users of computer software to understanding how software actually works from the inside. Introduction to Programming is the foundation of everything we will do in the next four years, and learning it properly is essential for every engineering student.

### Moving Beyond Basic Computer Usage
Before starting this degree, computers were mostly about browsing the internet, using applications, or playing games. However, our initial classes completely changed this perspective. We began by learning about how a computer execution cycle works. A computer hardware cannot understand human language; it only understands binary code (0s and 1s). Programming acts as the bridge between human logic and machine execution. We learned how high-level languages are translated by compilers into machine code so that the processor can perform operations.

### The Core Logic of Programming
For a computer engineer, writing code is not just about memorizing syntax or typing commands in an editor. It is an architectural discipline. The real work happens in the mind before we even touch the keyboard. Programming forces you to look at a complex, chaotic real-world challenge and decompose it into pure, deterministic logic paths. For example, if we want to build a simple system to check user access, we have to think about every single possibility: What if the input is empty? What if the password is wrong? What if the system crashes? Handling these conditions taught me how to think structurally.

### Key Concepts Covered in Class
During the practical labs, we spent a lot of time implementing basic building blocks of programming. We started with variables and data types, understanding how integers, floats, and characters occupy different byte sizes in memory. Then we moved to conditional statements (if-else blocks), which allow the program to make decisions based on inputs. 

After that, we covered software loops (for loops and while loops). Loops are incredibly powerful because they allow us to repeat a specific block of logic thousands of times without rewriting the code. We also practiced writing functions to make our code modular and reusable, which is a key practice in professional software development.

### Learning to Think Like an Engineer
One of the biggest lessons I learned in the programming lab is that every single variable you declare represents a physical slot in the computer's RAM. If you write an inefficient loop or allocate memory poorly, it inflates the computational execution cost. An engineer’s job is not just to make the code work, but to make it work efficiently using minimum resources. This initial phase was tough because debugging logical errors takes time and patience, but it has laid down the unyielding mental framework required to approach scalable software engineering in my upcoming semesters.
