# Post 6: Relational Database Foundations and Structured Query Language

Databases serve as the absolute backbone of almost every modern software system, enterprise application, and digital platform in existence. In this academic module, I dove deep into the core fundamentals of Relational Database Management Systems, studying how data can be structured, linked, stored, and queried systematically without losing integrity.

Before this course, my idea of storing data was limited to simple text files or spreadsheets. However, spreadsheets quickly become chaotic and slow as information grows. A relational database organizes data into highly structured, two dimensional tables consisting of rows and columns. Each row represents a unique record, while each column represents a specific attribute. The true power of RDBMS lies in its ability to establish logical relationships across different tables, minimizing data duplication and ensuring that information remains consistent across the entire backend architecture.

During our practical lab assignments, we spent a significant amount of time learning how to construct these database schemas properly. The first step was understanding constraints, particularly Primary Keys and Foreign Keys. A Primary Key is a unique identifier assigned to a specific column that ensures no two rows in that table can be identical. A Foreign Key is a reference point placed in another table to build a secure logical link between them. We practiced writing exact Data Definition Language scripts to build tables, set up data types like integers and characters, and enforce these key constraints directly in the system.

Once our tables were structured and filled with data, we learned how to interact with them using Structured Query Language. Writing SQL queries taught me how to extract precise pieces of data from massive tables using filters, conditions, and logical operators. We practiced using the SELECT statement paired with WHERE clauses to filter records, and explored the power of relational algebra by joining multiple tables together to create comprehensive data views. This foundational understanding is vital for any engineer designing reliable software backends.
